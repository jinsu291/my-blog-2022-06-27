1. 자바 언어의 특징

- 자바응용프로그램은 운영체제나 하드웨어가 아닌 JVM하고만 통신하고 JVM이 자바응용프로그램으로부터 전달받은 명령을 해당 운영체제가 이해할 수 있도록 변환하여 전달
  -> Write one, run anywhere (한번 작성하면 어디서나 실행된다.)

- 객체지향개념의 특징인 상속, 캡슐화, 다형성이 잘 적용된 순수 객체지향언어

- 가비지 컬렉터가 자동적으로 메모리를 관리해줘 프로그래머가 메모리 따로 관리 필요x

- 자바에서 개발되는 멀티쓰레드 프로그램은 시스템과는 관계없이 구현가능, 여러 쓰레드에 대한 스케쥴링을 자바 인터프리터가 담당

- 자바는 동적 로딩을 지원 -> 실행 시 모든 클래스가 로딩되지 않고 필요한 시점에 클래스를 로딩하여 사용할 수 있다.

- 일부 클래스가 변경되어도 전체 애플리케이션을 다시 컴파일하지 않아도 됨.

- 애플리케이션의 변경사항이 발생해도 비교적 적은 작업만으로 처리할 수 있는 유연한 애플리케이션

2. 자바로 프로그램작성하기

- 자바에서 모든 코드는 반드시 클래스 안에 존재해야 하며, 서로 관련된 코드들을 그룹으로 나누어 별도의 클래스를 구성하게 된다.

- ‘public static void main(String[] args)’ 는 main 메서드의 선언부 -> 프로그램을 실행할 때 ‘java.exe’에 의해 호출될 수 있도록 미리 약속된 부분

//자바
pulic class Hello2 {}
pulic class Hello3 {}

- 하나의 소스파일에 둘 이상의 public class가 존재하면 안된다. 각 클래스를 별도의 소스 파일에 나눠서 저장하던가 아니면 둘 중의 한 클래스에 public을 붙이지 않아야한다.

- 범위주석 /\* \*/
  한줄 주석 //

//자바
System.out.println(“Hello, /_ 이것은 주석 아닌 _/ world”);
System.out.println(“Hello, world. //이것도 주석아님”);

- 문자열을 의미하는 큰따옴표(“) 안에 주석이 있을 때는 주석이 아닌 문자열로 인식

3. 에러메시지가 뜰 경우

- 자주 발생하는 경우 -> 변수 또는 메서드의 이름을 잘못 사용, ; 사용 x, {}의 개수가 맞지 않을 때 등

  ㄱ. 에러 메시지를 잘 읽고 해당 부분의 코드를 살펴본다. 이상이 없으면 해당 코드의 주위도 함께 살펴본다.

  ㄴ. 그래도 이상이 없으면 에러 메시지는 잊어버리고, 기본적인 부분을 재확인한다. 대부분의 에러는 사소한것인 경우가 많다.

  ㄷ. 의심이 가는 부분을 주석처리하거나 따로 떼어내서 테스트 한다.
