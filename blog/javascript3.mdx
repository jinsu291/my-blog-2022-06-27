title: "[Javascript 3]데이터 타입"
date: "2022-07-02"
---

- 모든 수를 실수로 처리하며, 정수만 표현하기 위한 데이터 타입이 별도로 존재하지 x

- 자바스크립트의 문자열은 원시타입이며, 변경 불가능한 값이다. 이것은 문자열이 생성되면 그 문자열을 변경할 수 없다는 것을 의미힌다.

- var 키워드로 선언한 변수는 암묵적으로 undefined로 초기화된다. 자바크립트 엔진이 변수를 초기화하는데 사용하는 defined를 개발자가 의도적으로 변수에 할당한다면 undefined 의본래 취지와 어긋날뿐더러 혼란을 줄수 있어 권장 x

- 변수에 값이 없다는 것을 명시하고 싶을 때는 null을 할당한다.

- 정적 타입언어(C, JAVA 등)는 변수의 타입을 변경할 수 없으며, 변수에 선언한 타입에 맞는 값만 할당할 수 있지만 자바스크립트는 정적 타입 언어와 다르게 변수를 선언할 때 타입을 선언하지 않는다. 이는 어떠한 데이터 타입의 값이라도 자유롭게 할당할 수 있다.

- 자바스크립트에서는 값을 할당하는 시점에 변수의 타입이 동적으로 결정되고, 변수의 타입을 언제든지 자유롭게 변경할 수 있다. 다시말해 자바스크립트의 변수는 선언이 아닌 할당에 의해 타입이 결정된다. 그리고 재할당에 의해 변수의 타입은 언제든지 동적으로 변할 수 있는데, 이를 동적타이핑이라 한다.

- 문제점으로는 변수 값은 언제든지 변경될 수 있기 떄문에 복잡한 프로그램에서는 변화하는 변수 값을 추적하기 어려울 수 있고,. 동적 타입 언어의 변수는 값을 확인하기 전에는 타입을 확신할 수 없다.

- 안정적인 프로그램을 만들기 위한 주의사항

1. 변수의 개수가 많으면 오류 확률이 높아져 변수의 무분별한 남발은 금물

2. 변수의 유효 범위는 최대한 좁게 만들어 변수의 부작용을 억제해야한다.

3. 전역변수는 최대한 사용하지 않는다.

4. 변수보다는 상수를 사용해 값의 변경을 억제한다.
