- 원시타입은 단 하나의 값만 나타내지만 객체타입은 다양한 타입의 값을 하나의 단위로 구성한 복합적인 자료구조이다. 또한 원시 타입의 값, 즉 원시 값은 변경 불가능한 값이지만 객체 타입의 값, 즉 객체는 변경 가능한 값이다.

- 객체는 0개 이상의 프로퍼치로 구성된 집합이며 프로퍼티는 키와 값으로 구성된다. 그리고 객체는 프로퍼티와 메서드로 구성된 집합체이다.

- 프로퍼티는 객체의 상태를 나타내는 값이고, 메서드는 프로퍼티(상태 데이터)를 참조하고 조작할 수 있는 동작이다.

- 함수로 객체를 생성하기도 하며, 함수 자체가 객체이니 서로 분리할 수 없는 개념이다.

- 자바스크립트는 프로토타입 기반 객체지향 언어로 다양한 객체 생성 방법을 지원한다.

//자스
var person = {
name : ‘Lee’ ,
sayHello : function() {
console.log(‘Hello! My name is ${this.name}.’);
}
};
console.log(typeof person); //object
console.log(person); //{name: “Lee”, sayHello: f}

- 객체 생성 방법 중에 가장 일반적이고 간단한 방법은 객체 리터럴을 사용하는 방법인데, 객체리터럴은 객체를 생성하기 위한 표기법이다.

- 객체 리터럴의 중괄호는 코드 블록을 의미하지 않는다는데 주의하자. 코드 블록의 닫는 중괄호 뒤에는 세미콜론을 붙이지 않는다. 하지만 객체 리터럴은 값으로 평가되는 표현식이니 닫는 중괄호 뒤에는 세미콜론을 붙인다.

//자스
var person = {
firstName : ‘Ung-mo’, //식별자 네이밍 규칙을 준수하는 프로퍼티 키
‘last-name’ : ‘Lee’ //식별자 네이밍 규칙을 준수하지 않는 프로퍼티 키
};
console.log(person);

- 프로퍼티 키는 포르퍼티 값에 접근할 수 있는 이름으로서 식별자 역할을 한다.

- 식별자 네이밍 규칙을 준수하는 이름, 즉 자바스크립트에서 사용 가능한 유효한 이름인 경우 따옴표를 생략할 수 있ㄷ. 반대로 말하면 식별자 네이밍 규칙을 따르지 않는 이름에는 반드시 따옴표를 사용해야한다.

//자스
var obj = {};
var key = ‘hello’;
obj[key] = ‘world’;
console.log(obj); //{hello: “world”}

- 문자열 또는 문자열로 평가할 수 있는 표현식을 사용해 프로퍼티 키를 동적으로 생성할 수 있다. 이 경우에는 프로퍼티 키로 사용할 표현식을 대괄호([...])로 묶어야한다.

//자스
var foo = {
name : ‘Lee’,
name : ‘Kim’
};
console.log(foo); //{name: “Kim”}

- 이미 존재하는 프로퍼티 키를 중복 선언하면 나중에 선언한 프로퍼티가 먼저 선언한 프로퍼티를 덮어쓴다. 이때 에러가 발생하지 않으니 주의할 것!!!

- 프로퍼티 값이 함수일 경우 일반 함수와 구분하기 위해 메서드라 부른다.

//자스
var person = {
name : ‘Lee’
};
console.log(person.name); //Lee
console.log(person[‘name’]); //Lee

- 대괄호 표기법을 사용하는 경우 대괄호 프로퍼티 접근 연산자 내부에 지정하는 프로퍼티 키는 반드시 따옴표로 감싼 문자열이어야 한다.

- 객체에 존재하지 않는 프로퍼티에 접근하면 undefined를 반환한다. 이때 ReferenceError가 발생하지 않는데 주의하자!!

//자스
var person = {
name : ‘Lee’
};
person.age = 20;
console.log(person); //{name: “Lee”, age : 20}

- 존재하지 않는 프로퍼티에 값을 할당하면 프로퍼티가 동적으로 생성되어 추가되고 프로퍼티 값이 할당된다.

var person = {
name : ‘Lee’
};
person.age = 20;
delete person.age;
console.log(person); //{name : “Lee”}

- delete 연산자는 객체의 프로퍼티를 삭제한다.
